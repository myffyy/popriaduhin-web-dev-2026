# Лабораторная работа № В-2

## Назначение

Проект реализует веб-калькулятор на PHP. Он умеет принимать строковое арифметическое выражение, вычислять его и показывать результат прямо на странице без использования `eval()`.

Калькулятор поддерживает:
- целые числа;
- десятичные дроби;
- операции `+`, `-`, `*`, `/`, `:`;
- круглые скобки `(` и `)`;
- хранение истории вычислений в сессии PHP.

## Возможности

- вычисление выражений с учетом приоритета операций;
- вычисление вложенных скобок рекурсивно;
- вывод понятных сообщений об ошибках;
- сохранение истории в рамках одной сессии браузера;
- добавление результата в историю сразу после вычисления.

## Как запустить

1. Разместить файлы проекта в директории веб-сервера с поддержкой PHP.
2. Открыть `index.php` в браузере через локальный сервер.
3. Ввести выражение в поле ввода.
4. Нажать кнопку `Вычислить`.

## Примеры ввода

- `2+2`
- `10-3*2`
- `(2+3)*4`
- `12.5/2`
- `8:(2+2)`

## Принцип работы

1. При загрузке страницы вызывается `session_start()`.
2. Если сессионные данные еще не созданы, инициализируется массив `$_SESSION['history']`.
3. На странице отображается форма с одним текстовым полем `val`.
4. После отправки формы введенная строка передается в `calculateSq()`.
5. `calculateSq()` обрабатывает выражение со скобками:
   - проверяет корректность расстановки скобок;
   - находит внутреннее выражение в самых левых скобках;
   - вычисляет его рекурсивно;
   - подставляет полученный результат обратно в строку;
   - повторяет процесс, пока скобки не закончатся.
6. После удаления скобок вычисление передается в `calculate()`.
7. `calculate()` разбирает выражение без скобок и считает его с учетом приоритета операций.
8. Если вычисление успешно, результат выводится на страницу и сразу добавляется в историю.
9. Если выражение некорректно, выводится сообщение об ошибке.

## Реализованные функции

- `isnum($x)` - проверяет, является ли значение числом;
- `calculate($val)` - вычисляет выражение без скобок;
- `SqValidator($val)` - проверяет правильность расстановки скобок;
- `calculateSq($val)` - вычисляет выражение со скобками;
- `session_start()` - включает работу сессии и позволяет хранить историю вычислений.

## Обработка выражений

Внутри `calculate()` операции обрабатываются в два этапа:

1. Сначала ищутся `+` и `-`.
2. Затем обрабатываются `*`, `/` и `:`.

Это нужно для соблюдения стандартного приоритета операций.

Функция `calculateSq()` работает рекурсивно. Она сначала вычисляет самую внутреннюю часть скобок, подставляет результат и повторяет процесс, пока в строке не останется скобок.

## Работа с историей

История хранится в `$_SESSION['history']`.

Каждая запись имеет вид:

```text
выражение = результат
```

Например:

```text
2+2 = 4
(2+3)*4 = 20
```

История сохраняется в пределах одной сессии браузера, то есть при обновлении страницы или повторной отправке формы записи не теряются.

## Особенности реализации

- не используется `eval()`;
- для проверки корректности входных данных применяются отдельные функции;
- предусмотрена обработка пустого выражения;
- отдельно обрабатываются ошибки скобок;
- деление на ноль возвращает понятное сообщение;
- результат вычисления попадает в историю сразу после успешного расчета.

## Ограничения

- калькулятор работает только с простыми арифметическими выражениями;
- не поддерживаются функции вроде `sin()`, `cos()`, `pow()`;
- не поддерживаются переменные и присваивания;
- ввод должен быть записан в допустимом формате без лишних символов.

## Контрольные вопросы

1. Вопрос: Как изменится работа функции из листинга В-2.4, если из нее убрать проверку на пустую строку?

   Ответ: Функция начнет пытаться обрабатывать пустую строку как выражение. Это приведет к неверному разбору и ошибочным результатам.

2. Вопрос: Что такое рекурсия в программировании? Приведите примеры рекурсии.

   Ответ: Рекурсия - это прием, при котором функция вызывает саму себя. Примеры: вычисление выражений со скобками, факториал числа, обход дерева.

3. Вопрос: Что такое база рекурсии?

   Ответ: База рекурсии - это условие, при котором рекурсивная функция завершает работу без дальнейших вызовов самой себя.

4. Вопрос: Почему математические действия в функции `calculate()` выполняются именно в такой последовательности?

   Ответ: Потому что в математике операции имеют разный приоритет: сначала выполняются умножение и деление, затем сложение и вычитание.

5. Вопрос: Что такое сессия в РНР?

   Ответ: Сессия в PHP - это механизм хранения данных пользователя между запросами в рамках одного сеанса работы браузера.

6. Вопрос: Почему нельзя начинать инициализировать данные сессии после любого вывода?

   Ответ: Потому что `session_start()` работает через HTTP-заголовки, а заголовки должны быть отправлены до любого вывода HTML или текста.

7. Вопрос: Можно ли инициализировать данные сессии внутри тега `<body>`, но при этом до вывода данных средствами РНР?

   Ответ: Формально можно, если до этого не было никакого вывода. Но правильнее вызывать `session_start()` в самом начале файла.

8. Вопрос: Как РНР разделяет пользователей для доступа к разным хранилищам сессий?

   Ответ: PHP использует уникальный идентификатор сессии. Он хранится в cookie или передается через URL, если cookies недоступны.

9. Вопрос: Как получить доступ к данным сессии из РНР?

   Ответ: Через суперглобальный массив `$_SESSION`.

10. Вопрос: Как долго можно обращаться к данным сессии?

    Ответ: Пока сессия активна. Обычно это длится до закрытия браузера или до истечения времени жизни сессии на сервере.

11. Вопрос: Можно ли хранить в сессии массивы и строки больше 1024 символов?

    Ответ: Да, можно, если это допускается конфигурацией сервера и размером хранилища сессий.

12. Вопрос: Как записать данные в сессию?

    Ответ: Присваиванием значения элементу массива `$_SESSION`, например `$_SESSION['key'] = $value;`.

13. Вопрос: Можно ли прочитать данные из сессии, если они туда не записаны? Как быть в том случае, если такая ситуация возможна?

    Ответ: Напрямую читать нельзя без проверки. Нужно сначала проверять существование элемента через `isset()`, чтобы избежать предупреждений.

14. Вопрос: Как быстро вычислить любое выражение средствами РНР?

    Ответ: Самый быстрый способ - использовать `eval()`, но в этой лабораторной работе его применять нельзя.

15. Вопрос: Можно ли в РНР интерпретировать символ строки как число?

    Ответ: Да, PHP может интерпретировать строковый символ как число в арифметических операциях или при явном преобразовании типов, но для корректной проверки ввода это нужно контролировать отдельно.
