# Лабораторная работа № А-3: Виртуальная клавиатура

## Описание программы

Эта программа представляет собой реализацию виртуальной цифровой клавиатуры с помощью PHP. Вся функциональность основана на передаче состояния через GET-параметры в URL, без использования сессий, cookie или файлов на стороне сервера.

**Основные возможности:**
- **Ввод цифр:** При нажатии на цифровые кнопки (0-9) соответствующая цифра добавляется в строку вывода.
- **Сброс:** Кнопка "СБРОС" очищает строку вывода.
- **Счетчик нажатий:** В нижней части страницы отображается общее количество нажатий на цифровые кнопки с момента последней перезагрузки страницы или сброса.

Состояние (текущая строка и количество нажатий) полностью управляется путем его передачи от страницы к странице в виде GET-параметров.

## Контрольные вопросы

### 1. Как передать GET-параметр через ссылку?

GET-параметр передается в URL-адресе ссылки (атрибут `href` тега `<a>`). После пути к файлу ставится знак вопроса `?`, за которым следует пара "имя_параметра=значение".
**Пример:** `<a href="index.php?param=value">Ссылка</a>`

### 2. Что такое $_GET?
`$_GET` — это суперглобальный ассоциативный массив в PHP. Он содержит все параметры, которые были переданы скрипту через URL методом GET. Ключами этого массива являются имена параметров.

### 3. Как передать в ссылке несколько параметров?
Несколько GET-параметров перечисляются в URL через символ амперсанда `&`.
**Пример:** `<a href="index.php?param1=value1&param2=value2">Ссылка</a>`

### 4. Как проверить существование переменной?
С помощью встроенной языковой конструкции `isset()`. Она возвращает `true`, если переменная была определена и её значение не равно `null`.
**Пример:** `if (isset($my_variable)) { ... }`

### 5. Как узнать, был ли передан параметр в скрипт?
Для этого нужно проверить, существует ли в массиве `$_GET` ключ с именем этого параметра. Это также делается с помощью `isset()`.
**Пример:** `if (isset($_GET['id'])) { ... }`

### 6. Как узнать значение переданного в сценарий параметра?
Нужно обратиться к элементу массива `$_GET` по его ключу (имени параметра). Рекомендуется всегда предварительно проверять его существование с помощью `isset()`, чтобы избежать ошибок.
**Пример:** `$userId = $_GET['id'];`

### 7. Изменится ли работа программы А-3.1, если между двумя условными операторами поставить else? Если да - то как?
Да, изменится.
- **Оригинальный код:** `if(...) ...; if(...) ...;` — оба условия проверяются независимо. Если передать параметры `num` и `reset` одновременно, то сначала к строке добавится цифра, а затем строка сразу же будет очищена.
- **С `else`:** `if(...) ...; else if(...) ...;` — если первое условие истинно, второе проверяться не будет. Если передать `num` и `reset` одновременно, выполнится только первый блок: цифра добавится, а сброс не произойдет.

### 8. Изменится ли работа программы А-3.4, если между двумя условными операторами убрать else? Если да - то как?
Нет, работа программы не изменится. В исходной логике `else` служит для того, чтобы предотвратить добавление `key` к только что инициализированной пустой строке `store`. Однако, при первой загрузке (когда `store` не задан) параметр `key` также отсутствует, поэтому второе условие `if(isset($_GET['key']))` и так будет ложным. Логика работы останется идентичной.

### 9. Изменится ли работа программы А-3.4, если убрать второй условный оператор, оставив else? Если да - то как?
Да, изменится, и программа станет работать некорректно.
- **Код станет таким:** `if(!isset($_GET['store'])) { ... } else { $_GET['store'] .= $_GET['key']; }`
- **Проблема:** Блок `else` будет выполняться всегда, когда `store` передан. Но если при этом `key` не был передан (например, при простом обновлении страницы), PHP сгенерирует ошибку `Notice: Undefined index: key`, так как будет предпринята попытка доступа к несуществующему элементу массива. Код станет нестабильным.
