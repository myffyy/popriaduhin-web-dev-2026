## 3. Реализация динамических элементов

Для выполнения требований лабораторной работы статический контент был заменен на PHP-код в следующих элементах:

### 3.1. Динамический заголовок `<title>`

На каждой странице заголовок, отображаемый во вкладке браузера, не является статичным. Он создается с помощью переменной `$pageTitle`, которой присваивается нужное значение, после чего она выводится внутри тега `<title>`.

**Пример кода:**

```php
<title><?php $pageTitle = "Главная страница"; echo $pageTitle; ?></title>
```

### 3.2. Динамический подвал `<footer>`

В подвале каждой страницы выводится актуальная дата и время ее формирования. Это реализовано с помощью функции `date()` в PHP, которая форматирует текущие серверные дату и время в заданный вид.

**Пример кода:**

```php
<footer>
    <p>Сформировано: <?php echo date("d.m.Y в H-i:s"); ?></p>
</footer>
```

### 3.3. Динамические таблицы

Таблицы на страницах также были сделаны динамическими согласно требованиям:

1.  **Первая строка таблицы** (`<thead>`): Весь HTML-код первой строки, включая теги `<tr>` и `<th>`, генерируется одной командой `echo` в PHP.
2.  **Вторая строка таблицы** (`<tbody>`): Во второй строке теги `<tr>` и `<td>` остаются статичными, а PHP используется только для вывода содержимого ячеек между ними.

Это демонстрирует два разных подхода к генерации контента, как и требовалось в задании. Промежуточные переменные для хранения текстовых строк не использовались.

**Пример кода:**

```php
// Генерация первой строки
echo "<tr><th>Динамический Заголовок 1</th>...</tr>";

// Генерация ячейки во второй строке
<td><?php echo "Дин. Строка 1, Ячейка 1"; ?></td>
```

### 3.4. Динамическое меню

Меню было реализовано строго в соответствии с учебными требованиями методического указания. На каждой странице (`index.php`, `page2.php`, `page3.php`) в HTML-код встроена статическая структура меню. Активный пункт меню выделяется с помощью класса `selected_menu`, который прописан у соответствующей ссылки в коде каждой страницы. Этот подход напрямую реализует требование по созданию уникальной навигации для каждой страницы без использования сложных функций, что позволяет продемонстрировать базовое понимание структуры HTML и CSS.

**Пример кода на `index.php`:**

```html
<ul>
	<li><a href="index.php" class="selected_menu">Главная</a></li>
	<li><a href="page2.php">Вторая страница</a></li>
	<li><a href="page3.php">Третья страница</a></li>
</ul>
```

### 3.5. Динамические изображения

Для создания динамического эффекта на странице изображение меняется каждую секунду. Это реализовано с помощью PHP:

- Функция `date('s')` получает текущую секунду.
- Оператор остатка от деления (`% 2`) определяет, является ли секунда четной или нечетной.
- В зависимости от результата (`1` или `2`) формируется имя файла (`photo1.jpg` или `photo2.jpg`), которое подставляется в тег `<img>`.

**Пример кода:**

```php
<?php echo '<img src="photos/photo'.(date('s') % 2 + 1).'.jpg" alt="Меняющаяся фотография">'; ?>
```

## 6. Контрольные вопросы

**1. Что такое РНР?**

PHP (рекурсивный акроним от _PHP: Hypertext Preprocessor_) — это скриптовый язык программирования, разработанный специально для веб-разработки. Его код выполняется на стороне сервера, а браузеру пользователя отправляется результат выполнения, как правило, в виде HTML-разметки.

**2. Какое основное назначение PHP?**

Основное назначение PHP — создание динамических веб-страниц. Это включает в себя широкий спектр задач: обработка данных из HTML-форм, работа с базами данных (например, MySQL), управление сессиями и аутентификацией пользователей, генерация изображений, работа с файлами на сервере и многое другое, что делает сайты интерактивными.

**3. В чем отличие статического и динамического контента?**

- **Статический контент** — это готовые файлы (например, `.html`, `.css`, `.jpg`), которые хранятся на сервере и отдаются пользователю без изменений. Все пользователи видят одно и то же.
- **Динамический контент** — это содержимое, которое генерируется на сервере в момент запроса пользователя. Оно может быть разным для разных пользователей или меняться со временем, так как зависит от данных (например, из базы данных), действий пользователя или других условий.

**4. Как внедрить РНР-код на статическую страницу?**

PHP-код встраивается непосредственно в HTML-файл с помощью специальных тегов: `<?php ... ?>`. Весь код, заключенный в эти теги, будет выполнен на сервере. Чтобы сервер распознал и выполнил код, файл должен иметь расширение `.php`.

**5. Какие общепринятые требования к страницам с РНР кодом?**

- **Разделение логики и представления:** Стараться отделять PHP-код, отвечающий за логику (обработка данных, запросы к БД), от HTML-кода, отвечающего за отображение.
- **Безопасность:** Обязательно проверять и очищать любые данные, приходящие от пользователя, чтобы предотвратить уязвимости, такие как SQL-инъекции и XSS-атаки.
- **Читаемость:** Писать понятный код, использовать осмысленные имена переменных и оставлять комментарии для сложных участков.
- **Оптимизация:** Избегать "тяжелых" и ресурсоемких операций, которые могут замедлить загрузку страницы.

**6. Как трансформировать код вывода ссылки А-1.8 так, чтобы он не содержал статических фрагментов?**

Для этого необходимо всю HTML-структуру ссылки `<a>` формировать внутри PHP и выводить с помощью `echo`. Статические части (`<a href="`, `>` и `</a>`) конкатенируются (объединяются) с переменными.

**Пример:**

```php
<?php
$name = 'Вторая страница';
$link = 'page2.php';
$current_page = true;

// Вся строка формируется в PHP
$tag = '<a href="' . $link . '"';

if ($current_page) {
    $tag .= ' class="selected_menu"'; // Добавляем класс, если ссылка активна
}

$tag .= '>' . $name . '</a>'; // Добавляем текст и закрывающий тег

echo $tag; // Выводим готовую ссылку
?>
```

**7. Как будет выглядеть словесное описание алгоритмов, реализуемых в листингах А-1.5 ... А-1.8?**

- **Листинг А-1.5:** Алгоритм выводит одну заранее подготовленную HTML-строку, содержащую полностью сформированный тег ссылки.
- **Листинг А-1.6:** Алгоритм состоит из двух шагов: сначала выводится первая часть строки со ссылкой (до класса), а затем сразу вторая часть (класс, текст и закрывающий тег).
- **Листинг А-1.7:** Алгоритм демонстрирует "фрагментарное" внедрение PHP. Статические части HTML-кода (`<a href="`, `class="` и т.д.) чередуются с PHP-вставками, которые выводят только динамические значения (адрес, имя класса).
- **Листинг А-1.8:** Алгоритм сначала определяет переменные (имя, адрес, активность ссылки). Затем он начинает выводить тег `<a>`, используя переменные. С помощью условного оператора `if` он решает, нужно ли добавлять класс для активной ссылки. В конце он выводит имя ссылки и закрывающий тег.
