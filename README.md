# Лабораторная работа №7. Сортировка массивов

## Структура программы

Программа состоит из трёх файлов:

| Файл | Назначение |
|------|-----------|
| `index.php` | Форма ввода массива (HTML + JavaScript, без PHP-кода) |
| `sort.php` | Обработчик формы — валидация, сортировка, вывод результатов (PHP) |
| `styles.css` | Стили оформления страниц |

### index.php

Содержит статическую HTML-форму с:
- Таблицей `<table id="elements">` для динамического добавления полей ввода.
- Селектором `<select name="algoritm">` с шестью алгоритмами сортировки.
- Скрытым полем `<input type="hidden" id="arrLength">`, хранящим количество элементов.
- Тремя кнопками: «Добавить ещё один элемент», «Удалить последний элемент», «Сортировать массив».

**JavaScript-функции:**

- `addElement(table_name, amount)` — добавляет строку в таблицу с номером элемента и полем ввода. Имена полей формируются как `element0`, `element1`, ... `elementN`. Обновляет значение `arrLength`.
- `removeElement(table_name)` — удаляет последнюю строку таблицы (минимум 1 элемент остаётся).
- `setHTML(element, txt)` — кроссбраузерная установка innerHTML элемента.

При нажатии «Сортировать массив» данные формы отправляются методом POST в `sort.php`, который открывается в новой вкладке (`target="_blank"`).

### sort.php

Алгоритм работы обработчика:

1. Проверка наличия данных (`$_POST['element0']`). Если нет — сообщение и `exit()`.
2. Валидация всех элементов функцией `arg_is_not_Num()`. Если элемент не число — сообщение и `exit()`.
3. Определение выбранного алгоритма, вывод его названия.
4. Формирование и вывод исходного массива.
5. Засечка времени (`microtime(true)`).
6. Вызов функции сортировки с выводом состояния массива на каждой итерации.
7. Вычисление затраченного времени.
8. Вывод итога: количество итераций и время выполнения.

## Описание алгоритмов сортировки

### 1. Сортировка выбором (Selection Sort)

**Принцип:** на каждом шаге ищем минимальный элемент в неотсортированной части массива и ставим его на своё место.

**Пример:** `[3, 4, 2, 1]`

```
Итерация 1: ищем min в [3,4,2,1] → min=1 (индекс 3), меняем с [0]
  → [1, 4, 2, 3]

Итерация 2: ищем min в [4,2,3] → min=2 (индекс 2), меняем с [1]
  → [1, 2, 4, 3]

Итерация 3: ищем min в [4,3] → min=3 (индекс 3), меняем с [2]
  → [1, 2, 3, 4]

Результат: [1, 2, 3, 4]
```

**Сложность:** O(n²)

---

### 2. Пузырьковая сортировка (Bubble Sort)

**Принцип:** последовательно сравниваем соседние элементы и меняем местами, если они стоят в неправильном порядке. «Тяжёлые» элементы «тонут» вниз, «лёгкие» — «всплывают» наверх.

**Пример:** `[2, 4, 1, 3]`

```
Проход 1:
  Сравниваем [0] и [1]: 2 < 4 — не меняем   → [2, 4, 1, 3]
  Сравниваем [1] и [2]: 4 > 1 — меняем       → [2, 1, 4, 3]
  Сравниваем [2] и [3]: 4 > 3 — меняем       → [2, 1, 3, 4]

Проход 2:
  Сравниваем [0] и [1]: 2 > 1 — меняем       → [1, 2, 3, 4]
  Сравниваем [1] и [2]: 2 < 3 — не меняем    → [1, 2, 3, 4]

Проход 3:
  Сравниваем [0] и [1]: 1 < 2 — не меняем    → [1, 2, 3, 4]

Результат: [1, 2, 3, 4]
```

**Сложность:** O(n²)

---

### 3. Алгоритм Шелла (Shell Sort)

**Принцип:** улучшенная сортировка вставками. Сравниваются элементы не на соседних позициях, а на расстоянии `k`, которое постепенно уменьшается до 1. Последняя итерация — классическая сортировка вставками.

Последовательность расстояний: `N/2, N/4, ..., 1`

**Пример:** `[5, 3, 8, 1, 2]` (N=5)

```
k=3 (ceil(5/2)):
  Сравниваем элементы на расстоянии 3:
  [5,3,8,1,2] → сравниваем 1 и 5 → [1,3,8,5,2]
  [1,3,8,5,2] → сравниваем 2 и 3 → [1,2,8,5,3]

k=2 (ceil(3/2)):
  Сравниваем элементы на расстоянии 2:
  [1,2,8,5,3] → 8>1, ок → [1,2,8,5,3]
  [1,2,8,5,3] → 5>2, ок → [1,2,8,5,3]
  [1,2,8,5,3] → 3<8 → [1,2,3,5,8]

k=1:
  Обычная сортировка вставками — массив уже отсортирован.

Результат: [1, 2, 3, 5, 8]
```

**Сложность:** O(n log²n) в среднем

---

### 4. Алгоритм садового гнома (Gnome Sort)

**Принцип:** гном смотрит на текущий и предыдущий горшок. Если они в правильном порядке — шагает вперёд. Если нет — меняет местами и шагает назад. Начинает со второго элемента.

**Пример:** `[4, 2, 3, 1]`

```
i=1: arr[0]=4 > arr[1]=2 → меняем → [2,4,3,1], i=0
i=0: нет предыдущего → шагаем вперёд, i=2
i=2: arr[1]=4 > arr[2]=3 → меняем → [2,3,4,1], i=1
i=1: arr[0]=2 < arr[1]=3 → шагаем вперёд, i=3
i=3: arr[2]=4 > arr[3]=1 → меняем → [2,3,1,4], i=2
i=2: arr[1]=3 > arr[2]=1 → меняем → [2,1,3,4], i=1
i=1: arr[0]=2 > arr[1]=1 → меняем → [1,2,3,4], i=0
i=0: нет предыдущего → шагаем вперёд, i=4
i=4: выход из цикла

Результат: [1, 2, 3, 4]
```

**Сложность:** O(n²)

---

### 5. Быстрая сортировка (Quick Sort)

**Принцип:** выбирается опорный элемент (средний). Массив разделяется на две части: элементы меньше опорного — слева, больше — справа. Затем рекурсивно сортируются обе части.

**Пример:** `[4, 9, 7, 6, 2, 3, 8]`

```
Опорный = arr[3] = 6
Левая граница l=0, правая r=6

Итерация 1: сдвигаем l→ (4<6, 9≥6 → l=1), r← (8>6, 3≤6 → r=5)
  Меняем arr[1] и arr[5]: [4,3,7,6,2,9,8], l=2, r=4

Итерация 2: l→ (7≥6 → l=2), r← (2≤6 → r=4)
  Меняем arr[2] и arr[4]: [4,3,2,6,7,9,8], l=3, r=3

Итерация 3: l→ (6≥6 → l=3), r← (6≤6 → r=3)
  Меняем arr[3] и arr[3]: [4,3,2,6,7,9,8], l=4, r=2

Рекурсия: сортируем [4,3,2] и [7,9,8] по тому же принципу

Результат: [2, 3, 4, 6, 7, 8, 9]
```

**Сложность:** O(n log n) в среднем, O(n²) в худшем случае

---

### 6. Встроенная функция PHP — `sort()`

Использует алгоритм **Introsort** (комбинация Quick Sort, Heap Sort и Insertion Sort). Сортирует массив по возрастанию значений. Возвращает `true` при успехе, `false` — при ошибке. Количество итераций не отслеживается.

## Ответы на контрольные вопросы

### 1. Какие изменения необходимо внести, чтобы программа работала без скрытого поля arrLength?

Вместо `arrLength` можно на стороне PHP перебирать `$_POST` и считать элементы, имена которых начинаются с `element`:

```php
$arrLength = 0;
while( isset($_POST['element'.$arrLength]) ) {
    $arrLength++;
}
```

Таким образом, скрытое поле становится необязательным — длина массива определяется динамически.

### 2. Как при нажатии кнопки «Добавить ещё один элемент» добавлять не один, а несколько элементов сразу?

Изменить второй аргумент функции `addElement()` при вызове:

```html
<input type="button" value="Добавить 5 элементов"
    onClick="addElement('elements', 5);">
```

Функция `addElement` уже содержит цикл `for(var i=0; i<amount; i++)`, поэтому достаточно передать нужное количество.

### 3. Как будет выглядеть обратная к arg_is_not_Num() функция (возвращает true, если аргумент — число)?

```php
function arg_is_Num( $arg )
{
    if( $arg==='' ) return false;   // пустая строка — не число

    for($i=0; $i<strlen($arg); $i++)
        if( $arg[$i]!='0' && $arg[$i]!='1' && $arg[$i]!='2' &&
            $arg[$i]!='3' && $arg[$i]!='4' && $arg[$i]!='5' &&
            $arg[$i]!='6' && $arg[$i]!='7' && $arg[$i]!='8' &&
            $arg[$i]!='9' )
            return false;   // встретилась не цифра

    return true;    // все символы — цифры
}
```

Логика инвертирована: `return true` и `return false` поменялись местами.

### 4. Как работают алгоритмы шейкерной сортировки, сортировки слиянием и вставками?

**Шейкерная сортировка** — двунаправленная пузырьковая сортировка. Устанавливаются две границы (левая и правая). Сначала проход справа налево — самый «лёгкий» элемент всплывает в начало, левая граница сдвигается вправо. Затем проход слева направо — самый «тяжёлый» элемент тонет в конец, правая граница сдвигается влево. Цикл продолжается, пока границы не сойдутся.

**Сортировка вставками** — каждый элемент массива вставляется на своё место в уже отсортированную часть. Начиная со второго элемента, он сравнивается с предыдущими и сдвигает их вправо, пока не найдёт подходящую позицию.

**Сортировка слиянием** — массив рекурсивно делится пополам до элементарных частей (1 элемент). Затем части попарно сливаются в отсортированном порядке: из двух отсортированных половин поэлементно выбирается меньший и добавляется в результат.

### 5. В каком случае на листинге 7.9 без условия $j>=0 будет бесконечный цикл?

Если текущий вставляемый элемент меньше всех предыдущих, цикл `while` будет сдвигать `$j` всё дальше влево. Без проверки `$j>=0` индекс уйдёт в отрицательные значения, и цикл никогда не завершится (обращение к `$arr[-1]`, `$arr[-2]`, ...).

### 6. Как изменится работа программы листинга 7.9, если в цикле с предусловием будет $j>0?

Условие `$j>0` вместо `$j>=0` приведёт к тому, что элемент с индексом 0 никогда не будет сдвинут. Если вставляемый элемент должен встать на позицию 0 (он наименьший), он встанет на позицию 1, а `$arr[0]` останется на месте. Массив будет отсортирован неправильно.

### 7. Какой метод сортировки использует стандартная функция PHP?

Стандартная функция `sort()` в PHP использует алгоритм **Introsort** — гибридный алгоритм, комбинирующий Quick Sort (быстрая сортировка), Heap Sort (пирамидальная) и Insertion Sort (вставками). Переключается между ними в зависимости от размера данных и глубины рекурсии.

### 8. Как досрочно прекратить выполнение PHP-программы?

С помощью функции `exit()` (или её алиаса `die()`). Можно передать строку-сообщение: `exit('Ошибка');` — оно будет выведено перед завершением. Также можно передать целочисленный код завершения: `exit(1);`.

### 9. Если была вызвана функция exit() — будет ли в браузер передана статическая вёрстка, размещённая после последнего фрагмента кода PHP?

**Нет.** Функция `exit()` немедленно прекращает выполнение PHP и дальнейшую передачу данных в браузер. В браузер попадёт только то, что было выведено (статически или через PHP) **до** вызова `exit()`.

### 10. Как получить системное время с помощью PHP?

С помощью функции `microtime(true)`:

```php
$time = microtime(true);
```

Возвращает текущее время Unix в виде вещественного числа (секунды с точностью до микросекунд). Без аргумента `true` возвращает строку формата `"msec sec"`.

Также можно использовать `time()` (целые секунды) или `hrtime(true)` (наносекунды).

### 11. Какие параметры присутствуют у функции microtime() и для чего они используются?

Функция `microtime()` имеет один необязательный параметр `$as_float` (тип `bool`):

- **`microtime()`** или **`microtime(false)`** — возвращает строку вида `"0.25000000 1648000000"` (дробная часть секунд + целые секунды).
- **`microtime(true)`** — возвращает `float`, например `1648000000.25` — удобно для вычисления разницы во времени простым вычитанием.
