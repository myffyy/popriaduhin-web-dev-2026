## Алгоритм работы программы

### Общая архитектура

Сайт построен по модульному принципу. Единственная точка входа — `index.php`. Все остальные файлы подключаются как модули и не доступны напрямую через URL.

1. `index.php` подключает `menu.php` через `require` (меню нужно всегда).
2. В зависимости от GET-параметра `p` подключается соответствующий модуль контента через `include`:
   - `p=viewer` — просмотр (по умолчанию)
   - `p=add` — добавление
   - `p=edit` — редактирование
   - `p=delete` — удаление
3. Динамическое подключение: `if(file_exists($_GET['p'].'.php')) { include $_GET['p'].'.php'; }` — имя модуля формируется из значения параметра.

### Описание модулей

#### menu.php

- Проверяет наличие и допустимость параметра `p` (белый список: `viewer`, `add`, `edit`, `delete`). Если параметр отсутствует или недопустим — устанавливается `viewer`.
- Формирует 4 пункта меню в виде ссылок `<a>`. Активный пункт выделяется классом `selected` (красный цвет).
- Если выбран пункт «Просмотр» (`p=viewer`), дополнительно выводится подменю сортировки: «По-умолчанию» (`sort=byid`), «По фамилии» (`sort=fam`), «По дате рождения» (`sort=birth`).

#### index.php

- Подключает `menu.php` через `require`.
- Для режима `viewer`: подключает `viewer.php` (содержит функцию `getFriendsList`), валидирует параметры `pg` (страница) и `sort` (сортировка), вызывает функцию и выводит результат.
- Для остальных режимов: динамически подключает модуль по имени параметра `p`.

#### viewer.php — функция `getFriendsList($type, $page)`

- Подключается к БД MySQL (`localhost`, `root`, `''`, `friends`).
- Выполняет `SELECT COUNT(*)` для определения количества записей.
- Вычисляет количество страниц: `ceil($TOTAL / 10)`.
- Формирует SQL-запрос с `ORDER BY` (по `id`, `surname`+`name` или `birthday`) и `LIMIT` для пагинации (10 записей на страницу).
- Строит HTML-таблицу со всеми полями записей.
- Если страниц больше одной — формирует блок пагинации (ссылки на страницы, текущая выделяется `<span>`).
- Возвращает готовую HTML-строку.

#### add.php

- Содержит статическую HTML-форму со всеми полями записи (method="post", action="?p=add").
- При получении POST-данных (кнопка «Добавить запись»): подключается к БД, выполняет `INSERT INTO friends`. Все значения обрабатываются `htmlspecialchars()`.
- Выводит сообщение: «Запись добавлена» (зеленым) или «Запись не добавлена» (красным).

#### edit.php

- Подключается к БД. Если получены POST-данные — выполняет `UPDATE friends SET ... WHERE id=...`, приводя `id` к `(int)`.
- Определяет текущую запись: по GET-параметру `id`, иначе первая запись (ORDER BY surname, name).
- Выводит список всех записей (фамилия + имя) в виде ссылок. Текущая запись выделяется `<div>` вместо `<a>`.
- Выводит форму с полями, заполненными значениями текущей записи. `id` передается через скрытое поле `<input type="hidden">`.

#### delete.php

- Подключается к БД. Если передан GET-параметр `del` — сначала получает фамилию записи, затем выполняет `DELETE FROM friends WHERE id=...`.
- Выводит сообщение: «Запись с фамилией ... удалена» или ошибку.
- Выводит список всех записей (фамилия + инициалы) в виде ссылок для удаления.

#### styles.css

- Оформление меню: ссылки в виде кнопок синего цвета, активный пункт — красный (`class="selected"`).
- Подменю: уменьшенный шрифт, отделено верхней границей.
- Таблица: с рамками, фон заголовков серый.
- Пагинация: ссылки-кнопки, при наведении рамка утолщается до 2px, текущая страница — красным.
- Формы: белый фон, скругленные блоки, элементы в столбик.
- Сообщения: `.ok` — зеленый, `.error` — красный.

## Ключевые особенности

- **Модульность**: `require` для обязательного модуля (menu.php), `include` для динамических.
- **Безопасность**: белый список допустимых значений параметра `p`; `htmlspecialchars()` для экранирования ввода; приведение `id` к `(int)`.
- **Пагинация**: по 10 записей на страницу с пересчетом при выходе за границы.

## Контрольные вопросы

### 1. Что такое внешние модули? Зачем они нужны? Как они подключаются?

Внешние модули — это отдельные PHP-файлы с кодом, которые подключаются к основной программе. Они нужны для повторного использования кода, упрощения разработки и разделения логики. Подключаются конструкциями `include`, `include_once`, `require`, `require_once`.

### 2. Что такое библиотека функций? Зачем они нужны? Как они подключаются?

Библиотека функций — файл, содержащий набор пользовательских функций (например, `viewer.php` с функцией `getFriendsList`). Нужны для вынесения повторяющейся логики в отдельный модуль. Подключаются через `include`/`require`, после чего функции становятся доступны в основной программе.

### 3. В чем отличие конструкций include и require?

`include` добавляет код из файла во время выполнения программы. При ошибке (файл не найден) выдает предупреждение (Warning), но программа продолжает работу. `require` вставляет код из файла перед выполнением программы. При ошибке выдает фатальную ошибку (Fatal Error) и останавливает программу.

### 4. В чем отличие конструкций include и include_once?

`include` подключает файл каждый раз при вызове, даже если он уже был подключен ранее. `include_once` проверяет, был ли файл уже подключен, и если да — повторно его не подключает. Это предотвращает ошибки переопределения функций и классов.

### 5. В чем отличие конструкций require и require_once?

Аналогично паре include/include_once: `require` подключает файл каждый раз, `require_once` — только один раз. При повторном вызове `require_once` код файла не выполняется.

### 6. Что такое SQL?

SQL (Structured Query Language) — язык структурированных запросов для работы с реляционными базами данных. Позволяет выполнять поиск, фильтрацию, добавление, удаление и редактирование данных.

### 7. Основные операторы языка SQL?

- `SELECT` — выборка записей
- `INSERT INTO` — добавление записей
- `UPDATE` — изменение записей
- `DELETE FROM` — удаление записей
- `CREATE TABLE/DATABASE` — создание таблиц/баз данных
- `DROP TABLE/DATABASE` — удаление таблиц/баз данных
- `TRUNCATE TABLE` — очистка таблицы

### 8. Оператор INSERT. Для чего он предназначен, какие формы может иметь?

Предназначен для добавления записей в таблицу. Формы:

- `INSERT INTO table VALUES (val_1, ...)` — значения по порядку столбцов
- `INSERT INTO table (col_1, ...) VALUES (val_1, ...)` — с явным указанием столбцов
- `INSERT INTO table SELECT ... FROM another_table` — вставка из результата выборки

### 9. Источники данных для оператора INSERT: как добавить записи из другой таблицы?

`INSERT INTO table SELECT col_1, col_2 FROM another_table WHERE ...` — результат SELECT вставляется в целевую таблицу. Типы и количество столбцов должны совпадать.

### 10. Оператор UPDATE: назначение, синтаксис, форма записи?

Изменяет значения полей в существующих записях. Синтаксис: `UPDATE table_name SET field_1 = value_1, ... WHERE condition`. Без WHERE изменятся все записи таблицы. Пример: `UPDATE users SET name="Петров", age=19 WHERE id=1`.

### 11. Формы оператора SELECT?

`SELECT field_names FROM table_names WHERE conditions GROUP BY group_fields ORDER BY order_fields LIMIT rows, offset`. Все части кроме SELECT и FROM необязательны.

### 12. Условие отбора в операторе SELECT?

Задается после `WHERE`. Может содержать: имена полей, математические операторы (`+`, `-`, `*`, `/`), логические операторы (`AND`, `OR`, `NOT`), операторы сравнения (`=`, `<>`, `<`, `>`, `<=`, `>=`), скобки, функции.

### 13. Функции в операторе SELECT?

- `COUNT(*)` — количество записей
- `MAX(field)` — максимальное значение
- `MIN(field)` — минимальное значение
- `SUM(field)` — сумма значений
- `AVG(field)` — среднее значение
- `CONCAT()` — конкатенация строк
- `ROUND()`, `FLOOR()`, `CEILING()` — округление

### 14. Сортировка записей и ее типы?

Задается через `ORDER BY field`. Типы: `ASC` — по возрастанию (по умолчанию), `DESC` — по убыванию. Можно сортировать по нескольким полям: `ORDER BY age DESC, name ASC`.

### 15. Группировка записей: назначение и отличие от сортировки?

`GROUP BY` объединяет записи с одинаковыми значениями указанного поля в одну строку результата. Используется совместно с агрегатными функциями (COUNT, MAX, MIN, AVG, SUM). Сортировка (`ORDER BY`) лишь упорядочивает записи, не объединяя их.

### 16. Работа SQL-функций с группировкой и без группировки?

Без `GROUP BY` агрегатная функция применяется ко всем записям таблицы и возвращает одну строку. С `GROUP BY` функция применяется отдельно к каждой группе, возвращая по строке на группу. Пример: `SELECT name, MAX(age) FROM users GROUP BY name` — максимальный возраст для каждого имени.

### 17. Ограничение результата SQL-запроса: способы и параметры?

`LIMIT rows, offset` — `rows` задает максимальное количество возвращаемых записей, `offset` — отступ от начала. Пример: `LIMIT 10, 20` — пропустить 10/20 записей, вернуть следующие 20/10 (в зависимости от контекста: `LIMIT offset, count`).

### 18. Псевдонимы полей в операторе SELECT?

Задаются через `AS`: `SELECT name AS fio FROM users`. Переименовывают поле в результирующей таблице. Не изменяют исходную таблицу.

### 19. Расширение MySQLi — назначение и способ применения?

MySQLi — расширение PHP для доступа к серверу MySQL. Поддерживает процедурный и объектно-ориентированный стили. Основные функции: `mysqli_connect()`, `mysqli_query()`, `mysqli_fetch_assoc()`, `mysqli_errno()`, `mysqli_close()` и др.

### 20. Отличие процедурного и объектно-ориентированного стиля в MySQLi?

Процедурный: функции вида `mysqli_connect($host, ...)`, `mysqli_query($mysqli, $sql)` — объект подключения передается первым аргументом. ООП: методы объекта `$mysqli = new mysqli(...)`, `$mysqli->query($sql)` — вызываются через оператор `->`.

### 21. Буферизованные и не буферизованные запросы в MySQLi?

Буферизованный (`MYSQLI_STORE_RESULT`, по умолчанию) — весь результат загружается в память PHP сразу; можно узнать количество строк, выполнять другие запросы. Не буферизованный (`MYSQLI_USE_RESULT`) — данные получаются построчно с сервера; экономит память, но блокирует соединение до полного чтения.

### 22. Функции и методы расширения MySQLi?

- `mysqli_connect()` / `new mysqli()` — подключение
- `mysqli_connect_errno()` / `$mysqli->connect_errno` — код ошибки подключения
- `mysqli_query()` / `$mysqli->query()` — выполнение запроса
- `mysqli_fetch_row()` / `$res->fetch_row()` — строка как список
- `mysqli_fetch_assoc()` / `$res->fetch_assoc()` — строка как ассоциативный массив
- `mysqli_num_rows()` / `$res->num_rows` — количество строк
- `mysqli_errno()` / `$mysqli->errno` — код ошибки запроса
- `mysqli_affected_rows()` — количество затронутых строк
- `mysqli_close()` / `$mysqli->close()` — закрытие соединения

### 23. Что такое PDO?

PDO (PHP Data Objects) — универсальное расширение PHP для работы с любым сервером баз данных (MySQL, PostgreSQL, SQLite и др.). Использует объектно-ориентированный подход. Поддерживает подготовленные запросы с псевдопеременными, обработку ошибок через исключения (try/catch).

### 24. Чем отличаются расширения PDO и MySQLi?

MySQLi работает только с MySQL. PDO — универсальное, работает с любой СУБД. PDO поддерживает только ООП стиль. MySQLi — процедурный и ООП. PDO использует исключения для обработки ошибок. Оба поддерживают подготовленные запросы.

### 25. В чем отличие именованных и неименованных псевдопеременных в PDO?

Неименованные обозначаются `?` и привязываются по порядковому номеру: `bindParam(1, $var)`. Именованные обозначаются `:name` и привязываются по имени: `bindParam(':name', $var)`. Именованные нагляднее при большом количестве параметров.

### 26. Способы передачи значений в псевдопеременные запроса?

1. Через `bindParam()` — привязка переменной по ссылке.
2. Через массив в `execute()` — `$sth->execute(array($val1, $val2))` для неименованных или `$sth->execute(array(':name'=>$val))` для именованных.

### 27. Какова цель и преимущества использования псевдопеременных в SQL?

Цель — защита от SQL-инъекций и оптимизация повторных запросов. Преимущества: автоматическое экранирование символов; кэширование подготовленного запроса на сервере БД; уменьшение нагрузки при многократном выполнении однотипных запросов.

### 28. Как изменится работа программы, если в листинге В-1.1 в ссылках подменю не будет передаваться параметр "p"?

Подменю сортировки перестанет работать корректно: при клике на пункт подменю параметр `p` не будет передан, и в `menu.php` сработает проверка `if(!isset($_GET['p']))`, установив `p=viewer`. Визуально подменю исчезнет на один клик, но при следующей загрузке всё восстановится. Однако выделение основного пункта «Просмотр» может сброситься.

### 29. Почему в листинге В-1.4 при выводе пагинации в тексте ссылок страницы нумеруются начиная с единицы, а в адресе ссылок начиная с нуля?

Для пользователя нумерация с 1 более понятна (страница 1, 2, 3...). Внутри программы нумерация с 0 удобна для расчета диапазона `LIMIT`: `LIMIT ($page * 10), 10` — для страницы 0 это записи 0–9, для страницы 1 — записи 10–19 и т.д.

### 30. В каких случаях имеет смысл делать отдельный SQL-запрос для определения текущей записи?

Когда выводятся не все записи (например, при пагинации) — текущая запись может не попасть в выборку. Также когда записи содержат много данных (комментарии, фотографии) — загрузка всех записей целиком будет затратной по ресурсам. Отдельный запрос позволяет получить полную информацию только о нужной записи.

### 31. Как изменится работа программы в листинге В-1.11, если внутренний блок try-catch вынести в отдельную функцию?

Поведение не изменится — исключение 'A' будет перехвачено внутри функции в её catch-блоке, исключение 'B' будет выброшено из функции и перехвачено внешним catch. Результат останется тем же: «1 2 A 4 B 6». Единственное отличие — стек вызовов в `getTrace()` будет содержать вызов функции.

### 32. Как будет выглядеть строка программы на листинге В-1.11, если во вложенный try-catch добавить finally, выводящий символ «Z»?

Блок `finally` выполнится после внутреннего catch (после строки 10), перед тем как исключение 'B' покинет внутренний блок. Результат: «1 2 A 4 B Z 6» — символ Z появится между строками 10 и 12, т.к. finally гарантированно выполняется после try/catch вне зависимости от исключений.
