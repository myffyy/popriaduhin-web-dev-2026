# Лабораторная работа №8. Основы работы со строковыми данными в PHP. Кодировка. Анализ текста.

## Цель работы

Получение представлений об организации хранения текста в PHP, принципов работы с ним, особенностей обработки текстовой информации в различной кодировке.

## Структура проекта

```
ЛР_8/
├── index.html   — статический HTML-файл с формой ввода текста
├── result.php   — PHP-скрипт анализа текста
└── README.md    — описание проекта
```

## Описание файлов

### index.html

Статическая HTML-страница в кодировке UTF-8. Содержит:

- Заголовок страницы
- Форму с полем `<textarea>` для ввода многострочного текста
- Кнопку «Анализировать», отправляющую данные методом POST на `result.php`

### result.php

PHP-скрипт, принимающий текст из формы и выполняющий его анализ. Логика работы:

1. **Проверка входных данных** — если текст не передан или пуст, выводится сообщение «Нет текста для анализа».
2. **Вывод исходного текста** — текст отображается цветом и курсивом в блоке `<div class="src_text">`.
3. **Перекодировка** — перед анализом текст конвертируется из UTF-8 в CP1251 с помощью `iconv()`, чтобы каждый символ занимал ровно 1 байт и стандартные строковые функции PHP работали корректно.
4. **Анализ текста** (функция `test_it()`):
   - Подсчёт количества символов (`strlen()`)
   - Подсчёт букв (строчных и заглавных отдельно) через ассоциативные массивы
   - Подсчёт знаков препинания через ассоциативный массив
   - Подсчёт цифр через ассоциативный массив
   - Подсчёт слов — посимвольный перебор, разделителями служат пробелы и знаки препинания
   - Формирование списка слов с количеством вхождений (слова приводятся к нижнему регистру)
5. **Подсчёт вхождений символов** (функция `test_symbs()`) — текст приводится к нижнему регистру, затем посимвольно подсчитывается количество каждого уникального символа.
6. **Вывод результатов** — все данные оформляются в виде таблиц с границами. Перед выводом строки перекодируются обратно из CP1251 в UTF-8.
7. **Кнопка «Другой анализ»** — ссылка `<a>` на `index.html`.

## Контрольные вопросы

### 1. Какие стандартные функции сортировки массивов есть в PHP и как они работают?

- `sort()` — сортировка массива по значениям в порядке возрастания (индексы переназначаются).
- `rsort()` — сортировка по значениям в порядке убывания.
- `asort()` — сортировка по значениям с сохранением ключей.
- `arsort()` — сортировка по значениям в порядке убывания с сохранением ключей.
- `ksort()` — сортировка по ключам в порядке возрастания.
- `krsort()` — сортировка по ключам в порядке убывания.
- `usort()`, `uasort()`, `uksort()` — сортировка с использованием пользовательской функции сравнения.

### 2. Как проверить была ли определена функция?

С помощью функции `function_exists()`:

```php
if (function_exists('mb_strtolower')) {
    // функция существует
}
```

Возвращает `true`, если функция с указанным именем определена, и `false` в противном случае.

### 3. Как изменить регистр строки?

- `strtolower($str)` — переводит строку в нижний регистр.
- `strtoupper($str)` — переводит строку в верхний регистр.
- `ucfirst($str)` — делает первый символ строки заглавным.
- `lcfirst($str)` — делает первый символ строки строчным.
- `ucwords($str)` — делает заглавным первый символ каждого слова.

Для multibyte-строк используются аналоги с префиксом `mb_`: `mb_strtolower()`, `mb_strtoupper()`.

### 4. Что такое функции для multibyte-строк и в чём их отличие от стандартных?

Multibyte-функции (с префиксом `mb_`) корректно работают с многобайтовыми кодировками, такими как UTF-8, где один символ может занимать от 1 до 4 байт. Стандартные строковые функции PHP (`strlen`, `substr`, `strtolower` и т.д.) считают, что каждый символ — это 1 байт, поэтому для кириллицы в UTF-8 они возвращают неверные результаты. Например, `strlen("Привет")` вернёт 12 (а не 6), а `mb_strlen("Привет", "utf-8")` — корректные 6.

### 5. Что такое кодировка?

Кодировка — это таблица соответствия между символами (буквами, цифрами, знаками) и их числовыми кодами, которые используются для хранения и передачи текста в компьютере. Кодировка определяет, какой байт (или последовательность байт) соответствует какому символу.

### 6. Какие кодировки Вы знаете? В чём их отличия? Преимущества и недостатки?

| Кодировка | Размер символа | Особенности |
|-----------|---------------|-------------|
| **ASCII** | 1 байт (7 бит) | 128 символов. Только латиница и базовые знаки. Не поддерживает кириллицу. |
| **CP1251 (Windows-1251)** | 1 байт | 256 символов. Поддерживает кириллицу. Преимущество: каждый символ = 1 байт, простая работа со строками. Недостаток: ограничен 256 символами, нельзя одновременно использовать символы разных языков. |
| **UTF-8** | 1–4 байта | Поддерживает все символы Unicode (более 1 млн). Преимущество: универсальность, обратная совместимость с ASCII. Недостаток: переменная длина символов усложняет посимвольную обработку стандартными функциями. |
| **UTF-16** | 2–4 байта | Используется внутри Windows и Java. Преимущество: большинство символов — 2 байта. Недостаток: не совместима с ASCII, больший объём для латинского текста. |
| **KOI8-R** | 1 байт | Кириллическая кодировка, популярная в Unix-системах. Устарела. |

### 7. Как перекодировать строку в PHP из одной кодировки в другую?

С помощью функции `iconv()`:

```php
$result = iconv("utf-8", "cp1251", $str); // из UTF-8 в CP1251
$result = iconv("cp1251", "utf-8", $str); // из CP1251 в UTF-8
```

Синтаксис: `iconv($in_charset, $out_charset, $str)` — принимает исходную кодировку, целевую кодировку и строку. Возвращает перекодированную строку.

Также можно использовать `mb_convert_encoding()`:

```php
$result = mb_convert_encoding($str, "UTF-8", "CP1251");
```
