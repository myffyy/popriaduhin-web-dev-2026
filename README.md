# Лабораторная работа №6 — Тест математических знаний

## Описание

Веб-приложение на PHP, реализующее тест математических знаний. Представляет собой одну HTML-страницу (`index.php`), которая выводит форму, обрабатывает данные и отображает результат. Стили вынесены в отдельный файл `style.css`.

## Структура проекта

```
ЛР_6/
├── index.php        — основной файл (форма + обработчик)
├── style.css        — стили оформления
├── composer.json     — зависимости (PHPMailer)
├── vendor/           — установленные пакеты (автогенерация)
└── README.md         — описание проекта
```

## Установка

```bash
composer install
```

## Доступные задачи

| Значение (`TASK`) | Задача                 | Формула                                                        |
| ----------------- | ---------------------- | -------------------------------------------------------------- |
| `triangle_area`   | Площадь треугольника   | Формула Герона: `S = √(p(p-A)(p-B)(p-C))`, где `p = (A+B+C)/2` |
| `perimetr`        | Периметр треугольника  | `P = A + B + C`                                                |
| `volume`          | Объём параллелепипеда  | `V = A * B * C`                                                |
| `mean`            | Среднее арифметическое | `M = (A + B + C) / 3`                                          |
| `sum_squares`     | Сумма квадратов        | `S = A² + B² + C²`                                             |
| `hypotenuse`      | Гипотенуза             | `H = √(A² + B²)`                                               |

## Алгоритм работы программы

### 1. Первая загрузка страницы (GET-запрос)

```
Начало
  │
  ├─ Проверка: isset($_POST['A']) → НЕТ
  │
  ├─ Генерация случайных значений A, B, C (от 0 до 100, дробные)
  │
  ├─ Проверка GET-параметров (?F=...&G=...) для заполнения ФИО и группы
  │
  └─ Вывод HTML-формы со всеми элементами
```

### 2. Отправка формы (POST-запрос, нажата кнопка "Проверить")

```
Начало
  │
  ├─ Проверка: isset($_POST['A']) → ДА
  │
  ├─ Извлечение значений A, B, C из $_POST
  │
  ├─ Замена запятых на точки (поддержка дробей)
  │
  ├─ Приведение к float
  │
  ├─ Определение типа задачи по $_POST['TASK']
  │    ├─ 'mean'          → $result = round((A+B+C)/3, 2)
  │    ├─ 'triangle_area' → формула Герона
  │    ├─ 'perimetr'      → $result = A+B+C
  │    ├─ 'volume'        → $result = round(A*B*C, 2)
  │    ├─ 'sum_squares'   → $result = round(A²+B²+C², 2)
  │    └─ 'hypotenuse'    → $result = round(√(A²+B²), 2)
  │
  ├─ Формирование отчёта в $out_text:
  │    ├─ ФИО и группа
  │    ├─ Информация «о себе» (если заполнена)
  │    ├─ Тип задачи
  │    ├─ Входные данные (A, B, C)
  │    ├─ Предполагаемый результат (или «задача не решена»)
  │    ├─ Вычисленный результат
  │    └─ Вердикт: «ТЕСТ ПРОЙДЕН» / «ОШИБКА: ТЕСТ НЕ ПРОЙДЕН!»
  │
  ├─ Вывод отчёта:
  │    ├─ VERSION == 'print' → простой вид (без кнопки повтора)
  │    └─ VERSION == 'browser' → полный вид + кнопка «Повторить тест»
  │
  ├─ Если установлен флажок send_mail:
  │    ├─ Отправка письма через PHPMailer (SMTP, smtp.yandex.ru:465)
  │    └─ Вывод сообщения «Результаты отправлены на e-mail ...»
  │
  └─ Конец
```

### 3. Повторное тестирование

При нажатии ссылки «Повторить тест» формируется GET-запрос вида `?F=ФИО&G=Группа`. Форма загружается заново с новыми случайными значениями A, B, C, но поля ФИО и группа заполняются из GET-параметров.

### 4. JavaScript

Функция `toggleMail()` показывает/скрывает блок с полем e-mail при установке/снятии флажка, изменяя `style.display` элемента `#mail_block`.

## Ответы на контрольные вопросы

### 1. Что такое форма?

Форма (HTML-форма) — это раздел HTML-документа, содержащий элементы управления (поля ввода, кнопки, списки и т.д.), которые позволяют пользователю вводить данные и отправлять их на сервер для обработки. Форма задаётся тегом `<form>`.

### 2. Какие типы форм бывают?

Формы различаются по методу передачи данных:

- **GET-формы** — данные передаются в URL-строке (в адресной строке браузера).
- **POST-формы** — данные передаются в теле HTTP-запроса (не видны в URL).

Также формы могут различаться по типу кодирования (атрибут `enctype`):

- `application/x-www-form-urlencoded` — по умолчанию;
- `multipart/form-data` — для загрузки файлов;
- `text/plain` — простой текст.

### 3. В чём отличие POST- и GET-форм?

| Характеристика    | GET                                    | POST                          |
| ----------------- | -------------------------------------- | ----------------------------- |
| Передача данных   | В URL (строка запроса)                 | В теле HTTP-запроса           |
| Видимость         | Данные видны в адресной строке         | Данные скрыты от пользователя |
| Ограничение длины | Ограничена длиной URL (~2048 символов) | Практически не ограничена     |
| Кэширование       | Запросы кэшируются                     | Не кэшируются                 |
| Закладки          | Можно сохранить в закладки             | Нельзя                        |
| Безопасность      | Менее безопасен (данные в URL)         | Более безопасен               |
| PHP-массив        | `$_GET`                                | `$_POST`                      |

### 4. Какие элементы формы Вы знаете?

- `<input type="text">` — текстовое поле;
- `<input type="password">` — поле пароля;
- `<input type="checkbox">` — флажок;
- `<input type="radio">` — переключатель;
- `<input type="submit">` — кнопка отправки;
- `<input type="reset">` — кнопка сброса;
- `<input type="hidden">` — скрытое поле;
- `<input type="file">` — выбор файла;
- `<textarea>` — многострочное текстовое поле;
- `<select>` + `<option>` — выпадающий список;
- `<button>` — кнопка;
- `<input type="number">`, `<input type="email">`, `<input type="date">` и другие HTML5-типы.

### 5. Как передаёт своё значение каждый элемент формы в PHP-программу?

Каждый элемент формы имеет атрибут `name`. При отправке формы браузер формирует пары `имя=значение` для всех элементов и передаёт их серверу. В PHP эти данные доступны через суперглобальные массивы:

- `$_GET['имя']` — для GET-форм;
- `$_POST['имя']` — для POST-форм;
- `$_REQUEST['имя']` — объединённый массив.

Флажки (`checkbox`) передаются только если они отмечены. Селекторы (`select`) передают `value` выбранной опции.

### 6. Как округлить вещественное число до определённого количества разрядов после запятой?

С помощью функции `round()`:

```php
$rounded = round($number, 2); // округление до 2 знаков после запятой
```

Также можно использовать `number_format()` для форматирования вывода:

```php
$formatted = number_format($number, 2, '.', ''); // "3.14"
```

### 7. Сколько символов может максимально содержать строка, в которой буферизуется HTML-код?

В PHP строка может содержать до 2 ГБ данных (2 147 483 647 байт, ограничение `INT_MAX` на 32-битных системах). На 64-битных системах ограничение ещё выше. Фактический предел определяется доступной оперативной памятью и настройкой `memory_limit` в `php.ini`.

### 8. Как отправить сообщение по электронной почте средствами PHP?

С помощью встроенной функции `mail()`:

```php
mail($adress, $subject, $message, $headers);
```

Для работы функции на сервере должен быть настроен почтовый агент (sendmail, postfix и т.д.).

В данном проекте вместо `mail()` используется библиотека **PHPMailer** с отправкой через SMTP (Яндекс):

```php
$mail = new PHPMailer(true);
$mail->isSMTP();
$mail->Host       = 'smtp.yandex.ru';
$mail->SMTPAuth   = true;
$mail->Username   = 'логин@yandex.ru';
$mail->Password   = 'пароль_приложения';
$mail->SMTPSecure = PHPMailer::ENCRYPTION_SMTPS;
$mail->Port       = 465;
$mail->send();
```

Это позволяет отправлять письма без настройки локального почтового сервера.

### 9. Почему в листинге А-6.5 при сравнении переданного и вычисленного результата используется оператор эквивалентности (`===`)?

Оператор `===` (строгое сравнение) сравнивает и значение, и тип. Это важно, потому что:

- Оператор `==` выполняет приведение типов, из-за чего `0 == ""` вернёт `true`, хотя пользователь не вводил ответ.
- Если пользователь оставил поле пустым, `$_POST['result']` равно `""`. При использовании `==` пустая строка была бы равна `0`, и тест был бы ошибочно пройден при правильном ответе `0`.

Оператор `===` предотвращает такие ложные совпадения.

### 10. На какую функцию можно заменить `array_key_exists()` в листинге А-6.6?

Можно заменить на `isset()`:

```php
// Было:
if( array_key_exists('send_mail', $_POST) )

// Стало:
if( isset($_POST['send_mail']) )
```

Разница: `isset()` вернёт `false`, если значение равно `null`, а `array_key_exists()` — `true`. Но для элементов формы значение всегда не `null` (для checkbox это `"on"`), поэтому замена корректна.

### 11. Можно ли в листинге А-6.6 буферизовать сообщение об отправке письма в переменной `$out_text`?

Да, можно. Для этого нужно добавить сообщение в `$out_text` до вызова `echo`:

```php
if( array_key_exists('send_mail', $_POST) )
{
    mail( $_POST['MAIL'], 'Результат тестирования',
        str_replace('<br>', "\r\n", $out_text),
        "From: auto@mami.ru\n"."Content-Type: text/plain; charset=utf-8\n" );

    $out_text .= 'Результаты теста были автоматически отправлены на e-mail '.$_POST['MAIL'];
}

echo $out_text; // выводим весь отчёт одним вызовом
```

Такой код будет более оптимальным, так как:

- Уменьшается число вызовов `echo` (один вместо нескольких).
- Весь вывод хранится в одной переменной и может быть повторно использован (для письма и для браузера).
- Однако нужно следить за порядком: сообщение об отправке надо добавлять в `$out_text` **после** вызова `mail()`, чтобы в письме не было текста об отправке самого себя.

### 12. Какие события могут обрабатываться в JavaScript? Какими способами можно задать обработчик события для объекта?

**Основные группы событий:**

- **События мыши:** `click`, `dblclick`, `mousedown`, `mouseup`, `mousemove`, `mouseover`, `mouseout`
- **События клавиатуры:** `keydown`, `keyup`, `keypress`
- **События формы:** `submit`, `reset`, `change`, `focus`, `blur`, `input`
- **События документа/окна:** `load`, `unload`, `resize`, `scroll`, `DOMContentLoaded`
- **Сенсорные события:** `touchstart`, `touchmove`, `touchend`

**Способы назначения обработчика:**

1. **HTML-атрибут (инлайн):**

```html
<input type="checkbox" onClick="toggleMail(this)" />
```

2. **Свойство DOM-элемента:**

```javascript
document.getElementById('myBtn').onclick = function () {
	alert('Клик!')
}
```

3. **Метод `addEventListener()` (рекомендуемый):**

```javascript
document.getElementById('myBtn').addEventListener('click', function () {
	alert('Клик!')
})
```

Преимущество `addEventListener` — возможность назначить несколько обработчиков на одно событие и гибко управлять ими (удалять через `removeEventListener`).
