# Лабораторная работа № А-4: Пользовательские функции

Этот проект представляет собой решение лабораторной работы по теме "Пользовательские функции в PHP". Скрипт `index.php` генерирует HTML-страничку с несколькими таблицами, данные для которых берутся из специально отформатированных строк.

## Описание работы программы

Основная логика программы заключена в файле `index.php`.

### Принцип работы

1.  **Инициализация**: В начале скрипта задаётся переменная `$numberOfColumns`, которая определяет, сколько колонок должно быть во всех таблицах. Также инициализируется массив `$structures`, содержащий строки-шаблоны для создания таблиц.
2.  **Формат данных**: Каждая строка в массиве `$structures` описывает одну таблицу. Данные в строке имеют формат `"C1*C2#C3*C4"`, где:
    *   `*` — разделитель колонок (ячеек).
    *   `#` — разделитель строк.
3.  **Генерация таблиц**: Скрипт итерирует по массиву `$structures` и для каждого элемента вызывает функцию `outTable()`, которая отвечает за вывод одной полной таблицы.
4.  **Функциональное разделение**:
    *   `outTable($structure, $tableNumber, $columnCount)`: Принимает строку с полной структурой таблицы, её порядковый номер и требуемое число колонок. Она выводит заголовок `<h2>`, разбивает структуру на строки и для каждой строки вызывает `getTR()`. В конце она обрамляет результат тегами `<table>`.
    *   `getTR($rowData, $columnCount)`: Принимает строку, описывающую один ряд таблицы, и требуемое число колонок. Функция генерирует HTML-код для тега `<tr>`, разбивая строку на ячейки `<td>`. Если ячеек в данных меньше, чем `$numberOfColumns`, функция добавляет пустые ячейки, чтобы выровнять таблицу.

### Примеры вывода и обработка особых случаев

Программа корректно обрабатывает различные входные данные:

*   **Стандартный вывод**: Для строки вида `'Клетка1*Клетка2#Клетка3*Клетка4'` будет построена таблица 2x2 (при условии `$numberOfColumns >= 2`).
*   **Выравнивание колонок**: Если для строки задано меньше ячеек, чем `$numberOfColumns`, недостающие ячейки будут созданы пустыми. Если задано больше — "лишние" данные будут проигнорированы.
*   **Пустая структура**: Если строка в `$structures` пустая (`''`), программа выведет:
    ```
    В таблице нет строк
    ```
*   **Структура без ячеек**: Если строка содержит только разделители (`'##'`), то есть строки есть, но в них нет данных, программа выведет:
    
    ```
    В таблице нет строк с ячейками
    ```
*   **Неверное число колонок**: Если переменная `$numberOfColumns` установлена в `0` или отрицательное значение, программа не будет строить таблицы и выведет сообщение:
    ```
    Неправильное число колонок
    ```

## Ответы на контрольные вопросы

1.  **Что такое пользовательская функция?**
    Пользовательская функция — это именованный блок кода, созданный программистом для выполнения определённой задачи. Её можно многократно вызывать из разных частей программы, чтобы избежать дублирования кода.

2.  **Как функция возвращает значение?**
    Функция возвращает значение с помощью ключевого слова `return`, за которым следует возвращаемое значение или выражение.

3.  **Можно ли вызвать из функции другую функцию?**
    Да, можно. Это стандартная практика для построения сложных программ.

4.  **Можно ли вызвать из функции эту же функцию?**
    Да, можно. Этот приём называется рекурсией.

5.  **Продолжит ли функция свою работу после выполнения инструкции `return`?**
    Нет. Выполнение инструкции `return` немедленно завершает работу функции и возвращает управление в то место, откуда функция была вызвана.

6.  **Сколько раз инструкция `return` может быть использована в теле функции?**
    Инструкция `return` может встречаться в коде функции несколько раз (например, в разных ветвях условного оператора `if`), но за один вызов функции будет выполнена только одна из них, после чего работа функции прекратится.

7.  **Сколько аргументов может быть передано функции?**
    В PHP теоретически нет строгого ограничения на количество аргументов, но на практике оно лимитировано объёмом доступной памяти.

8.  **Что-такое аргументы "по умолчанию"?**
    Это аргументы, для которых в объявлении функции указано значение по умолчанию. Если при вызове функции для такого аргумента значение не передаётся, используется значение по умолчанию.

9.  **Может ли функция вообще не иметь аргументов?**
    Да, функция может быть объявлена без аргументов.

10. **Должны ли совпадать имена переменных-аргументов при объявлении и при вызове функции?**
    Нет, не должны. При вызове функции важны значения и их порядок, а не имена переменных, которые передаются в качестве аргументов.

11. **Если в теле функции объявлена переменная `$Р` — изменится ли эта переменная с таким же именем в основной программе? В другой функции? В этой же функции, но вызванной другой раз?**
    Нет, не изменится. Переменные, объявленные внутри функции, имеют локальную область видимости и существуют только во время её выполнения. Они не влияют на переменные с такими же именами за пределами функции или в других вызовах этой же функции.

12. **В каких случаях имеет смысл использовать пользовательские функции?**
    *   Когда один и тот же код нужно выполнить в нескольких местах (повторное использование).
    *   Для улучшения читаемости и структуры кода путём разбиения большой задачи на более мелкие и логически завершённые части.
    *   Для упрощения отладки и поддержки кода.

13. **Может ли функция выводить HTML-код?**
    Да, может. С помощью операторов `echo` или `print` функция может напрямую выводить HTML-разметку.

14. **Может ли функция вызываться непосредственно в операторе `echo`?**
    Да. Если функция возвращает значение (например, строку или число), `echo` выведет это значение.

15. **Может ли функция одновременно выводить текст в HTML-код браузера и возвращать значение?**
    Да. Функция может содержать операторы вывода (`echo`) и в конце иметь инструкцию `return` для возврата значения.

16. **В каком порядке формируется HTML-код, если функция не только вызывается непосредственно из оператора `echo`, но и выводит что-либо внутри себя?**
    Сначала выполнится код внутри функции и будут выведены все данные через `echo` или `print` внутри неё. Затем функция вернёт своё значение, и это значение будет выведено оператором `echo`, из которого она была вызвана.
    *Пример:* `echo "Старт " . my_func() . " Конец";` где `function my_func() { echo "Середина "; return "Возврат"; }` выведет: `Старт Середина Возврат Конец`.
